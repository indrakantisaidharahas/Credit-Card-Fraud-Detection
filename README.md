# PROJECT 5: Financial Fraud Detection

## 1. Background
Digital financial transactions have become a core part of modern economies. However, they are highly vulnerable to fraudulent activities such as unauthorized transactions and identity theft.  

Machine Learning (ML) techniques can help identify unusual patterns in transaction data and detect fraudulent behavior effectively.

---

## 2. Dataset
- **Name:** Credit Card Fraud Dataset  
- **Source:** Kaggle  
- **Link:** https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud  

---

## 3. Objective
The main objectives of this project are:
- Detect fraudulent transactions using machine learning models  
- Compare different detection approaches  
- Improve performance on highly imbalanced data  

---

## 4. Data Science Activities

### Data Cleaning
- Handle missing or inconsistent values  
- Remove duplicate records if necessary  

### Handling Imbalanced Data
- Apply techniques such as:
  - Undersampling  
  - Oversampling  
  - SMOTE (Synthetic Minority Oversampling Technique)  

### Feature Scaling
- Normalize or standardize features for better model performance  

### Model Development
- Train classification and anomaly detection models  

### Evaluation
- Evaluate models using appropriate performance metrics  

---

## 5. Required Machine Learning Tasks

### Classification Modeling
- Logistic Regression  
- Decision Trees  
- Random Forest  
- Support Vector Machines (SVM)  

### Anomaly Detection
- Isolation Forest  
- One-Class SVM  
- Autoencoders  

### Precision-Recall Evaluation
- Focus on precision and recall due to class imbalance  
- Use metrics like:
  - Precision  
  - Recall  
  - F1-score  
  - PR Curve  

### Model Comparison
- Compare models based on:
  - Accuracy (less important here)  
  - Precision  
  - Recall  
  - F1-score  
  - ROC-AUC
